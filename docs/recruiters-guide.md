# 🔎 Recruiter's Guide: Understanding the Candidate

Bienvenido. Si eres un IT Recruiter, Engineering Manager o CTO revisando este repositorio, **este documento es para ti**.

## El "Por Qué" de este Portafolio

La mayoría de los portfolios de desarrolladores son herramientas estáticas diseñadas para "mostrar código". Este portafolio está diseñado para demostrar **ejecución de producto, toma de decisiones arquitectónicas y madurez técnica**.

No estoy buscando presentarme genéricamente como "Full Stack". Mi perfil es:
> **Desarrollador enfocado en Backend (Sistemas y AI) con capacidades front-end modernas y una fuerte visión de Producto.**

## 🚀 De la v1 a la v2: Una historia de maduración

### v1 (El Pasado)
- **Stack:** Vite, React (SPA), CSS básico.
- **El problema:** Era un currículum digital rápido, pero no demostraba *qué sucede detrás del telón*. No había backend, no había persistencia, las métricas de rendimiento dependían fuertemente del cliente. Mostraba potencial, pero no maestría.

### v2 (El Presente)
- **Stack:** Next.js 16, React 19, Supabase (PostgreSQL), Tailwind 4, Arquitectura Cloud/Edge.
- **La solución:** Este proyecto es **un sistema complejo disfrazado de portafolio minimalista**. 
- Se construyó para responder a la pregunta: *"¿Puede este candidato diseñar una aplicación desde el diseño en Figma hasta el despliegue seguro en producción y conexión con bases de datos relacionales?"* La respuesta es sí.

## 💡 Lo que este código dice de mí:

1. **Entiendo la Arquitectura SSR/SSG (Server-Side Rendering)**
   - No me limito a hacer fetch de datos en el `useEffect`. Uso React Server Components y Server Actions para proteger credenciales y enviar menos JavaScript al navegador.

2. **No le temo a la Complejidad (Auth & DB)**
   - He integrado **Supabase Auth** con persistencia de cookies segura en lugar de usar mocks. Sé cómo asegurar rutas con Middlewares. Sé de Row Level Security (RLS) en Postgres.

3. **Valoro el "Pixel Perfect" y el Rendimiento**
   - El uso de **Tailwind CSS 4** y **Framer Motion** demuestra que puedo crear interfaces de usuario premium (como el efecto de 'Hover Preview' al cursor) asegurando 60fps, sin sacrificar la accesibilidad o el rendimiento de Lighthouse.

4. **Calidad de Código y Tipado Fuerte**
   - Todo el proyecto hace cumplir Strict `TypeScript` y utiliza validación de esquemas con `Zod`. Prevengo bugs en el servidor antes de que ocurran en runtime.
